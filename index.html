Rigorous evaluation for systems that operate in the real world.

A closer look at how I evaluate AI work.

I inspect whether a task says what it means, contains enough context, avoids accidental clues, and measures the intended capability.

What I look for +

Ambiguous verbs, missing output contracts, impossible constraints, hidden dependencies, reward hacking paths, weak examples, and mismatches between the prompt and verifier.

Why it matters +

I use oracle runs and verifier tests to check that valid approaches pass and shallow shortcuts fail for the right reason.

Verification notes +

I compare the natural-language requirement, reference solution, environment behavior, and automated tests as one evaluation system.

04 / Adversarial reviewFailure-mode map
Prompt injectionHallucinationData leakageUnsafe completionTool misuseOver-refusal

I probe beyond the happy path: unusual wording, malicious instructions, conflicting priorities, unsupported claims, and tool-use mistakes.

Testing approach +

Each test has a clear risk hypothesis, expected safe behavior, severity level, and evidence that another reviewer can audit.
